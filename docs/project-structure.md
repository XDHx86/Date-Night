# Project Structure

Understanding the project structure is key to navigating and contributing to the Datenight codebase. This document outlines the organization of the `src/` directory and explains the purpose of each major section.

## Overview

The application follows a modular, feature-centric structure with clear separation of concerns. All source code resides in the `src/` directory, which is organized as follows:

```
src/
├── assets/           # Static images, icons, and media files
├── components/       # Reusable UI components shared across routes
├── hooks/            # Custom React hooks encapsulating reusable logic
├── lib/              # Core application logic, utilities, and services
├── routes/           # Application pages (file-based routing)
├── router.tsx        # Router configuration (auto-generated route tree)
├── server.ts         # Server-side API routes and middleware
├── start.ts          # Application entry point
└── styles.css        # Global stylesheet and Tailwind CSS directives
```

## Detailed Breakdown

### `src/assets/`

Contains static assets such as:
- **Illustrations**: Themed images for different steps (begging.jpg, celebration.jpg, etc.)
- **Icons**: Any custom icon assets not served via Lucide
- **Media**: Audio files for sound effects (e.g., button clicks, confirmations)

### `src/components/`

Reusable, presentational components that are used across multiple routes. These components are typically dumb (presentational) components that receive props and do not manage application state directly.

Examples include:
- `AnimatedButton.tsx`: Custom button variants with hover/press animations
- `HeartBurst.tsx`: Celebratory particle effect used on confirmation screens
- `MovieCard.tsx`: Component for displaying movie information with gradient posters
- UI primitives from Radix (Dialog, Menu, Tooltip, etc.) wrapped in custom components
- Layout components like `PageShell.tsx` that provides consistent page structure

### `src/hooks/`

Custom React hooks that encapsulate reusable stateful logic. These hooks extract component logic into reusable functions.

Examples might include:
- Form handling hooks
- Animation control hooks
- Media query responders
- Data fetching wrappers (though data fetching is primarily handled by TanStack Query)

### `src/lib/`

The core application logic and services. This directory contains:

#### `store.ts`
- **Zustand store** managing application state
- Holds the selected date, time, and movie
- Provides actions to update each piece of state and reset the entire store
- Persists state to `sessionStorage` to survive page refreshes

#### `movies.ts`
- Movie data service containing:
  - Curated list of movies (`MOVIES` array)
  - Search functionality (`searchMovies(query)`)
  - Movie type definitions and helper functions
- Currently uses an offline-first approach with optional TMDB integration

#### `sound.ts`
- Audio management utility
- Handles loading and playing sound effects
- Provides a simple interface for triggering UI feedback sounds

#### `utils.ts` (if exists)
- Miscellaneous helper functions used across the application

### `src/routes/`

Application pages organized using TanStack Router's file-based conventions. Each file in this directory represents a route.

Key routes:
- `index.tsx`: Landing page (`/`) - the initial yes/no question
- `begging.tsx`: Playful "NO" button that dodges the cursor (`/begging`)
- `confirmation.tsx`: Celebration screen after "YES" (`/confirmation`)
- `date.tsx`: Date picker interface (`/date`)
- `time.tsx`: Time selector with quick options (`/time`)
- `movie.tsx`: Movie search and selection interface (`/movie`)
- `summary.tsx`: Review page showing selected date, time, and movie (`/summary`)
- `success.tsx`: Final celebration screen with details (`/success`)
- Supporting layout files:
  - `__root.tsx`: Application shell (providers, metadata, fonts)
  - `_layout.tsx`: Shared layout wrapper (if used)

> 💡 **Note**: The route tree is automatically generated in `routeTree.gen.ts` - **do not edit this file manually**.

### `src/router.tsx`

Exports the router configuration that ties together all route files. This file is typically generated by the TanStack Router plugin but may contain manual route definitions for advanced cases.

### `src/server.ts`

Defines server-side routes and API endpoints using Vinxi/Nitro under the hood. Currently used for:
- Potential API routes (if extending to use TMDB or other backend services)
- Middleware configuration
- Server-only utilities

### `src/start.ts`

The entry point of the application. Responsible for:
- Creating the Vinxi server instance
- Applying middleware
- Starting the development server or serving production builds

## Import Aliases

The project uses TypeScript path aliases configured in `tsconfig.json` for cleaner imports:

- `@/*`: Maps to `src/*` (e.g., `@/lib/store` instead of `../../lib/store`)
- `@/assets/*`: Maps to `src/assets/*`
- `@/components/*`: Maps to `src/components/*`
- And so on for other directories

These aliases are also configured in `vite.config.ts` via `vite-tsconfig-paths` to work in Vite.

## How to Add a New Feature

1. **Determine the feature type**:
   - UI component? → Add to `src/components/`
   - New page/route? → Add a file to `src/routes/`
   - Business logic? → Add to `src/lib/` (store, utils, or services)
   - Shared non-visual logic? → Consider a custom hook in `src/hooks/`
   - Styling/theme changes? → Edit `src/styles.css` or Tailwind config

2. **Follow existing patterns**:
   - Use TypeScript interfaces for props and state
   - Follow the existing component structure (props destructuring, early returns, etc.)
   - Use the Zustand store for shared state
   - Leverage Tailwind for styling
   - Use Framer Motion for animations
   - Handle errors gracefully with user-friendly messages

3. **Test your changes**:
   - Run the development server: `npm run dev`
   - Verify the feature works as expected
   - Check for linting errors: `npm run lint`
   - Ensure formatting is consistent: `npm run format`

## Finding Specific Elements

- **Looking for a button?** → Check `src/components/` for `AnimatedButton.tsx` or similar
- **Want to modify the date picker?** → Look at `src/routes/date.tsx`
- **Need to change how state is stored?** → Examine `src/lib/store.ts`
- **Looking to add a new animation?** → Review `framer-motion` usage in existing components
- **Trying to understand the data flow?** → Start with the Zustand store (`src/lib/store.ts`) and trace how components subscribe to and update state

## Documentation Conventions

Throughout the codebase, you'll find:
