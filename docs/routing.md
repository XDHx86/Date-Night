# Routing System

## Overview

Datenight uses **file-based routing** powered by [TanStack Router](https://tanstack.com/router/latest) via the TanStack Start framework. This means that the structure of the `src/routes/` directory directly defines the application's URL structure and navigation behavior.

## How It Works

- Each `.tsx` file inside `src/routes/` corresponds to a route.
- The file's path relative to `src/routes/` determines the URL path.
- Special filename conventions denote dynamic segments, optional parameters, and layouts.
- The router automatically generates a type-safe route tree (`routeTree.gen.ts`) that provides:
  - Autocompletion for navigation
  - Type safety for route parameters
  - Route validation at build time

> ⚠️ **Important**: `routeTree.gen.ts` is auto-generated. **Do not edit this file manually.**

## Route File Conventions

| File Pattern | Example | Route Path | Description |
|--------------|---------|------------|-------------|
| `index.tsx` | `src/routes/index.tsx` | `/` | The index route for a directory |
| `$folderName.tsx` | `src/routes/users/$id.tsx` | `/users/:id` | Dynamic parameter (required) |
| `{-$param}.tsx` | `src/routes/posts/{-$category}.tsx` | `/posts/:category?` | Optional parameter |
| `$$.tsx` | `src/files/$.tsx` | `/files/*` | Splatch (matches remaining path) |
| `_layout.tsx` | `src/routes/_layout.tsx` | (layout) | Wraps child routes; provides shared UI |
| `__root.tsx` | `src/routes/__root.tsx` | (root) | Application wrapper; provides providers, metadata, etc. |
| `(group)/` | `src/routes/(auth)/login.tsx` | `/login` | Route group (does not affect URL path) |

### Layout Routes
- `__root.tsx`: Application shell that wraps every page. Provides:
  - React Query client provider
  - Zustand store provider
  - Meta tags (title, description, OG tags)
  - Font imports and global styles
  - `<Outlet />` for child routes
- `_layout.tsx`: Not currently used in the base application but available for section-specific layouts (e.g., admin dashboard layout)

## Navigation

Navigation between pages is handled using TanStack Router's `useNavigate` hook or the `<Link>` component.

### Using the `useNavigate` Hook

```typescript
import { useNavigate } from '@tanstack/react-router';

function SomeComponent() {
  const navigate = useNavigate();

  const handleNext = () => {
    navigate({ to: '/time' }); // Navigates to the time picker
  };

  return (
    <button onClick={handleNext}>
      Continue to Time Selection
    </button>
  );
}
```

### Using the `<Link>` Component

```typescript
import { Link } from '@tanstack/react-router';

function NavigationMenu() {
  return (
    <nav>
      <Link href="/date">Select Date</Link>
      <Link href="/time">Choose Time</Link>
      <Link href="/movie">Pick a Movie</Link>
    </nav>
  );
}
```

### Programmatic Navigation with Options

You can also pass state or replace the current entry:

```typescript
navigate({
  to: '/summary',
  state: { from: 'direct-link' }, // Optional state object
  replace: true // Replace current history entry instead of pushing
});
```

## Route Parameters and Validation

While the current routes in Datenight don't use dynamic parameters, the pattern is supported for future expansion.

### Defining a Dynamic Route

To create a route with a parameter:
1. Create a file named `$paramName.tsx` (e.g., `product/$id.tsx`)
2. Access the parameter via the `useParams` hook:

```typescript
import { useParams } from '@tanstack/react-router';

function ProductDetail() {
  const { id } = useParams(); // { id: string }
  // Fetch product details using the id
}
```

### Optional Parameters

For optional parameters, use `{-$param}.tsx`:
- The parameter becomes optional in the URL
- The value will be `undefined` if not provided

### Splatch Routes

For paths that should match multiple segments (like a catch-all), use `$$.tsx`:
- Access the splat value via `_splat` parameter
- Commonly used for documentation sites or wildcard routes

## Route Loading and Error Handling

TanStack Router provides built-in mechanisms for:
- **Loading states**: Show spinners while route components load
- **Error boundaries**: Gracefully handle errors in route components
- **Pending navigation indicators**: Show progress during transitions

These features can be implemented using the router's built-in components or hooks, though the current implementation relies on component-level loading states.

## Adding a New Route

To add a new page to the application:

1. **Create the file**:
   - Determine the URL path (e.g., `/profile`)
   - Create the corresponding file: `src/routes/profile.tsx`
   - For nested paths: `src/routes/dashboard/analytics.tsx` → `/dashboard/analytics`

2. **Implement the component**:
   - Export a React component as the default export
   - Follow the existing patterns for layout, styling, and state usage
   - Access shared state via the Zustand store if needed
   - Use `useNavigate` or `<Link>` for navigation to/from this route

3. **Add navigation links**:
   - Update any menus, buttons, or flows that should access the new route
   - Use the `<Link>` component or `useNavigate` hook for type-safe navigation

4. **Test the route**:
   - Start the dev server: `npm run dev`
   - Navigate to the new URL manually or via links
   - Verify the component renders correctly and interacts with state as expected

5. **Update documentation** (optional but recommended):
   - Add the new route to this documentation
   - Include any relevant usage notes

## Route Safety and TypeScript

Thanks to TanStack Router's type generation:
- Navigating to a non-existent route results in a **compile-time error**
- Route parameters are strongly typed
- The `useMatch` hook provides accurate type inference for the current route
- IntelliSense in IDEs will suggest valid routes and parameters

## Examples from the Codebase

### Simple Navigation (from index.tsx to confirmation.tsx)

In `src/routes/index.tsx`:
```typescript
import { useNavigate } from '@tanstack/react-router';
import { AnimatedButton } from '@/components/AnimatedButton';

export default function Index() {
  const navigate = useNavigate();

  const handleYes = () => {
    navigate({ to: '/confirmation' });
  };

  return (
    <div className="flex flex-col items-center justify-center min-h-screen gap-6">
      <h1 className="text-3xl font-bold">Will you go out with me?</h1>
      <div className="flex gap-4">
        <AnimatedButton variant="ghost" onClick={handleYes}>
          YES
        </AnimatedButton>
        <AnimatedButton variant="outline" onClick={() => navigate({ to: '/begging' })}>
          NO
        </AnimatedButton>
      </div>
    </div>
  );
}
```

## Best Practices

1. **Keep routes focused**: Each route should represent a distinct view or step in the user flow.
2. **Leverage layout routes**: Use `_layout.tsx` or `__root.tsx` for shared UI elements (headers, footers, providers).
3. **Use error boundaries**: Consider wrapping route components in error boundaries for graceful degradation.
4. **Optimize loading**: For routes with heavy data fetching, consider using Tanstack Query's `useQuery` in route loader functions (if implementing server-side rendering).
5. **Stay consistent**: Follow the existing file naming conventions and component patterns.
6. **Document complex routes**: If a route has special behavior (e.g., requires authentication), add a comment explaining it.

## Troubleshooting

### "Route not found" errors
- Verify the file exists in `src/routes/` with the correct path
- Check that the filename follows the routing conventions
- Ensure the development server has been restarted after adding new files (Vite usually handles this automatically)

### TypeScript errors after adding a route
- Run `npm run dev` to trigger regeneration of `routeTree.gen.ts`
- Check your `tsconfig.json` for the correct `include` paths
- Ensure you're not trying to access a route that doesn't exist

### Links not working
- Verify you're using the `<Link>` component from `@tanstack/react-router`
- Check that the `href` prop matches a defined route exactly
- Ensure you're not accidentally using `<a>` tags for client-side navigation (use `<Link>` instead)
