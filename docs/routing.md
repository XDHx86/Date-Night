# Routing System

## Overview

Datenight uses **file-based routing** powered by [TanStack Router](https://tanstack.com/router/latest) via the TanStack Start framework. Each `.tsx` file inside `src/routes/` corresponds to a route, with the file's path determining the URL path.

## How It Works

- Each `.tsx` file inside `src/routes/` corresponds to a route
- The file's path relative to `src/routes/` determines the URL path
- Special filename conventions denote dynamic segments, optional parameters, and layouts
- The router automatically generates a type-safe route tree (`routeTree.gen.ts`)

> ⚠️ **Important**: `routeTree.gen.ts` is auto-generated. **Do not edit this file manually.**

## Route File Conventions

| File Pattern | Example | Route Path | Description |
|--------------|---------|------------|-------------|
| `index.tsx` | `src/routes/index.tsx` | `/` | The index route for a directory |
| `$folderName.tsx` | `src/routes/users/$id.tsx` | `/users/:id` | Dynamic parameter (required) |
| `{-$param}.tsx` | `src/routes/posts/{-$category}.tsx` | `/posts/:category?` | Optional parameter |
| `$$.tsx` | `src/files/$.tsx` | `/files/*` | Splatch (matches remaining path) |
| `_layout.tsx` | `src/routes/_layout.tsx` | (layout) | Wraps child routes; provides shared UI |
| `__root.tsx` | `src/routes/__root.tsx` | (root) | Application wrapper; provides providers, metadata, etc. |
| `(group)/` | `src/routes/(auth)/login.tsx` | `/login` | Route group (does not affect URL path) |

## Navigation

Navigation between pages is handled using TanStack Router's `useNavigate` hook or the `<Link>` component.

### Using the `useNavigate` Hook

```typescript
import { useNavigate } from '@tanstack/react-router';

function SomeComponent() {
  const navigate = useNavigate();

  const handleNext = () => {
    navigate({ to: '/time' }); // Navigates to the time picker
  };

  const handleBack = () => {
    navigate({ to: -1 }); // Goes back one entry in history
  };

  return (
    <button onClick={handleNext}>
      Continue to Time Selection
    </button>
  );
}
```

### Using the `<Link>` Component

```typescript
import { Link } from '@tanstack/react-router';

function NavigationMenu() {
  return (
    <nav>
      <Link href="/date">Select Date</Link>
      <Link href="/time">Choose Time</Link>
      <Link href="/movie">Pick a Movie</Link>
    </nav>
  );
}
```

### Programmatic Navigation with Options

```typescript
navigate({
  to: '/summary',
  replace: true // Replace current history entry instead of pushing
});
```

## URL Synchronization

**NEW in July 2026**: The application now has a centralized URL synchronization system that keeps Zustand state in sync with URL search parameters.

### The Problem (Before Fix)

Prior to the July 2026 fix, the application suffered from several navigation issues:

1. **`window.history.replaceState` Conflict**: The old `syncUrlWithState` function directly manipulated `window.history`, which bypassed TanStack Router's internal state management. This caused the router to be unaware of URL changes, leading to stale rendering.

2. **Duplicate Sync Logic**: Each route component created its own `syncUrlWithState()` instance, leading to:
   - Multiple independent `replaceState` calls
   - Race conditions between state updates and navigation
   - Stale closures capturing outdated state

3. **Render-Time Side Effects**: Some routes performed navigation directly in the render body (outside `useEffect`), causing:
   - Unpredictable timing of navigation calls
   - Components starting to render before navigating away
   - Race conditions during component mounting

### The Solution: `useUrlSync` Hook

A new centralized hook (`src/hooks/useUrlSync.ts`) was created to manage all URL synchronization:

```typescript
import { useUrlSync } from '@/hooks/useUrlSync';

function MyRouteComponent() {
  const { syncUrl, syncState } = useUrlSync();

  // Sync state from URL on mount
  useEffect(() => {
    syncState();
  }, [syncState]);

  // Sync URL when local state changes
  useEffect(() => {
    syncUrl();
  }, [someState, syncUrl]);

  return <div>...</div>;
}
```

### URL Parameter Format

The application synchronizes the following state to URL search parameters:

| State Property | URL Parameter | Format |
|----------------|---------------|--------|
| date | `date` | ISO format (yyyy-MM-dd) |
| time | `time` | 24-hour format (HH:mm) |
| movie | `movie` | Movie ID (number) |
| loveMessage | `love` | URL-encoded string |
| isDarkMode | `theme` | "dark" or omitted |

Example URL: `/movie?date=2026-07-15&time=19:30&movie=12345&theme=dark`

### Benefits

- ✅ Navigation works reliably on first click
- ✅ URL search params stay synchronized with Zustand state
- ✅ Browser back/forward navigation works correctly
- ✅ Refreshing any page correctly restores state from URL
- ✅ Progress bar updates match actual route
- ✅ No direct `window.history` manipulation
- ✅ Single source of truth for URL sync

## Progress Bar

The application features a persistent progress indicator (`TopProgressBar` component) that:

- Lives at the app root, outside of any route content
- Reads the active step directly from the URL (`useRouteStep`)
- Stays synchronized across navigation, refreshes, deep links, and browser back/forward
- Each step is a button that returns the user to that step's start
- Steps `<= current` are reachable; steps `> current` are blocked

### Progress Bar Steps

| Step | Route | Label | Icon |
|------|-------|-------|------|
| 1 | `/` | Start | Heart |
| 2 | `/date` | Date | CalendarHeart |
| 3 | `/time` | Time | Clock |
| 4 | `/movie` | Movie | Film |
| 5 | `/summary` | Summary | Sparkles |
| 6 | `/success` | Celebrate | PartyPopper |

> **Note**: The `/begging` and `/confirmation` routes are intentionally not included in the progress bar as they are part of the introduction flow.

## Route Guards

Some routes implement guards to ensure users follow the intended flow:

### Date Guard (time.tsx, movie.tsx, summary.tsx)

If a user deep-links to `/time` or `/movie` without having selected a date, they are redirected to `/date`:

```typescript
useEffect(() => {
  if (!date) {
    navigate({ to: "/date" });
  }
}, [date, navigate]);
```

> **Important**: This guard is now properly implemented in `useEffect` (not in render) to avoid render-time side effects.

### Selection Guard (summary.tsx)

The summary page ensures all required selections are made:

```typescript
useEffect(() => {
  if (!date || !time || !movie) {
    navigate({ to: "/date" });
  }
}, [date, time, movie, navigate]);
```

## Adding a New Route

To add a new page to the application:

1. **Create the file**:
   - Determine the URL path (e.g., `/profile`)
   - Create the corresponding file: `src/routes/profile.tsx`

2. **Implement the component**:
   - Export a React component as the default export
   - Follow the existing patterns for layout, styling, and state usage
   - Use `createFileRoute` for type safety:
     ```typescript
     import { createFileRoute } from '@tanstack/react-router';
     export const Route = createFileRoute('/profile')({ component: ProfilePage });
     ```

3. **Add URL synchronization** (if needed):
   - Import and use `useUrlSync` hook for state synchronization
   - Call `syncState()` on mount to hydrate from URL
   - Call `syncUrl()` when state changes

4. **Add navigation links**:
   - Update any menus, buttons, or flows that should access the new route
   - Use the `<Link>` component or `useNavigate` hook

5. **Update progress bar** (if applicable):
   - Add the route to `STEP_ROUTE` in `src/hooks/useRouteStep.ts`
   - Add the step definition to `STEP_DEFS`

6. **Test the route**:
   - Start the dev server: `npm run dev`
   - Navigate to the new URL manually or via links
   - Verify the component renders correctly and interacts with state as expected

7. **Update documentation** (optional but recommended):
   - Add the new route to this documentation
   - Include any relevant usage notes

## Route Safety and TypeScript

Thanks to TanStack Router's type generation:
- Navigating to a non-existent route results in a **compile-time error**
- Route parameters are strongly typed
- The `useMatch` hook provides accurate type inference for the current route
- IntelliSense in IDEs will suggest valid routes and parameters

## Navigation Best Practices

1. **Always use router APIs**: Use `navigate()` or `<Link>` instead of `window.history.pushState/replaceState`
2. **Avoid render-time side effects**: Place navigation calls in `useEffect`, not in the render body
3. **Use useUrlSync for state sync**: Don't create custom URL sync logic - use the centralized hook
4. **Batch state updates**: When multiple state values change, call `syncUrl()` once after all updates
5. **Preserve non-managed params**: The `useUrlSync` hook automatically preserves query params it doesn't manage (like UTM tags)

### ❌ Anti-Patterns

```typescript
// BAD: Direct history manipulation
window.history.pushState(null, '', newUrl);
window.history.replaceState(null, '', newUrl);
window.history.back();

// BAD: Navigation in render
if (!date) {
  navigate({ to: "/date" }); // This is in the render body!
}

// BAD: Creating sync function in render (stale closure)
const updateUrl = syncUrlWithState(); // New instance on each render
```

### ✅ Correct Patterns

```typescript
// GOOD: Use router navigate
navigate({ to: "/time" });
navigate({ to: -1 }); // Go back
// GOOD: Navigation in useEffect
useEffect(() => {
  if (!date) {
    navigate({ to: "/date" });
  }
}, [date, navigate]);

// GOOD: Use centralized hook
const { syncUrl, syncState } = useUrlSync();
useEffect(() => {
  syncState();
}, [syncState]);
useEffect(() => {
  syncUrl();
}, [state, syncUrl]);
```

## Troubleshooting

### "Route not found" errors
- Verify the file exists in `src/routes/` with the correct path
- Check that the filename follows the routing conventions
- Ensure the development server has been restarted after adding new files (Vite usually handles this automatically)

### TypeScript errors after adding a route
- Run `npm run dev` to trigger regeneration of `routeTree.gen.ts`
- Check your `tsconfig.json` for the correct `include` paths
- Ensure you're not trying to access a route that doesn't exist

### Links not working
- Verify you're using the `<Link>` component from `@tanstack/react-router`
- Check that the `href` prop matches a defined route exactly
- Ensure you're not accidentally using `<a>` tags for client-side navigation (use `<Link>` instead)

### Navigation not updating the page
- Ensure you're using `navigate()` from `@tanstack/react-router`, not `window.history`
- Check that navigation calls are in `useEffect`, not in render
- Verify you're using `useUrlSync` for URL synchronization, not custom history manipulation
